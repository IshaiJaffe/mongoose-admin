Mongoose-Admin

A front end to Mongoose - the ORM for Node.js and MongoDB

var AuthorModel = {
  name: {type: String, required:true},
  bio: {type: String}
};

var Author = new mongoose.Schema(AuthorModel);
mongoose.model('Author', Author);

var admin = mongoose-admin.createAdmin('mongodb://localhost/bookstore', 
                                       {port:8001});
admin.ensureUserExists('admin', 'my-secret-p@ssw0rd');
admin.registerModel('Author', AuthorModel, {list:['name'], sort['name']});

Features

  1. Automatic forms
  2. Support for nested objects
  3. Uses only public apis
  4. Support for linked objects
  5. Good test coverage

Options

The following options are available in the 2nd parameter of the createAdmin function:

  1. port: an integer value, present if the admin site should listen on a new port
  2. root: a string, present if the admin site should map to a path on the existing express app
  3. app: the existing express app to map into. This is required if root is supplied

The following options are available in the 3rd parameter of the registerModel function:

  1. list: an array of fields which should be display in views which list multiple objects of this kind
  2. sort: an array of fields which should be used as the default sort order in list views
  3. hidden: an array of fields which should be hidden from input forms
  4. pre: a function which should be called before any save happens. This function will be passed the object to be saved and is expected to return the object to be saved
  5. post: a function which should be called after any save happens. This function will be passed the object which was saved and is expected to return the object which was saved
